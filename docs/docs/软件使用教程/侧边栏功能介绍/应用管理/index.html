应用管理

本章节将介绍 ClassScreenLock 的应用管理功能，帮助您监控和管理受控计算机上的应用程序。

通过侧边栏的 “应用管理” 入口，您可以进入应用管理界面。该模块提供对受控计算机上所有已安装应用程序的集中监控与管理能力，包括实时进程监控与应用运行拦截。

界面总览​

进入应用管理后，您将看到如下主界面：

界面布局清晰，主要分为两个区域：

  • 左侧导航栏：用于切换不同的管理类别。
  • 右侧主面板：功能类似任务管理器，用于显示和管理对应类别的应用详细信息。

左侧导航栏包含三个核心功能模块：

功能详解​

1. 运行中的应用​

在此类别下，您可以实时监控所有当前正在运行的应用程序与进程。列表将清晰展示以下信息：

  • 应用状态（运行中/挂起等）
  • 内存占用（以 MB/GB 为单位）
  • 网络流量（上传/下载速度及累计用量）

通过此视图，您可以快速定位资源消耗异常、非授权软件或异常进程，并执行结束任务等管理操作。

2. 阻止运行的应用​

此功能允许您通过添加应用程序或文件的可执行路径（.exe），彻底禁止其在受控计算机上启动。

  • 拦截范围：不仅限于传统软件，也支持锁定其他文件类型（如 .mp3、.txt、.docx 等），实现灵活的管理策略。
  • 生效机制：被添加至列表的程序或文件，在用户尝试打开时系统将自动拦截并给出阻止提示。

⚖️ 法律与合规声明​

应用管理功能涉及对计算机进程的实时监控与对应用程序运行的强制干预，属于具有系统级控制能力的功能。使用本功能前，请务必严格遵守以下法律规定与合规原则：

一、授权与知情同意​

  • 设备授权：本功能仅可在自有设备或已获得设备所有人明确书面授权的受控设备上启用。未经许可擅自监控或拦截他人计算机中的应用，可能构成《刑法》第二百八十五条“非法控制计算机信息系统罪”。
  • 明示告知义务：在组织、学校或家庭等多人使用场景下部署本功能前，必须提前以显著方式告知所有可能受影响的用户：
    • 启用了应用监控与拦截策略；
    • 监控范围（如进程名称、资源占用）；
    • 拦截策略（如禁止运行的游戏/软件列表）；
    • 数据留存期限与申诉渠道。
  • 禁止隐蔽监控：严禁以“管理工具”为名行秘密监视之实。被监控用户有权知晓自己的设备活动是否被记录。

二、合法目的与禁止滥用​

  • 正当用途示例：
    • 学校在自有教学设备上屏蔽与课堂无关的游戏、影音软件；
    • 企业禁止员工在工作终端安装盗版或高风险软件；
    • 家长监护（须经未成年人监护人同意）限制特定应用使用时长；
    • IT 部门对异常进程进行临时隔离以处置安全事件。
  • 禁止行为：
    • ❌ 利用应用拦截功能恶意阻止竞争对手软件运行，构成不正当竞争；
    • ❌ 以限制用户自由为目的批量屏蔽合法常用软件（如浏览器、办公套件）；
    • ❌ 将进程监控信息用于员工隐私窥探、绩效歧视等非管理目的；
    • ❌ 协助他人规避软件许可验证或破解正版软件。

三、数据隐私与保护​

  • 本地优先原则：应用监控数据（如进程列表、内存占用）默认仅存储在本地，不会自动上传至云端。若您选择开启日志同步功能，必须另行获得被记录者的明确同意，并采取加密传输、访问控制等安全措施。
  • 最小必要采集：建议仅采集管理所必需的进程信息（如名称、PID、资源占用），避免记录窗口标题、浏览历史、键盘输入等高度敏感信息。
  • 日志处置：定期审查监控日志，对已过保存期限的数据进行彻底删除。若发生数据泄露事件，应立即启动应急预案并依法向监管机构报告。

四、责任边界与法律依据​

  • 用户自决原则：应用拦截规则完全由使用者自行配置，系统仅提供技术实现。使用者对自行创建的拦截策略、监控范围承担全部法律与合规责任。
  • 法律法规：本功能的使用须遵守《中华人民共和国网络安全法》《中华人民共和国数据安全法》《中华人民共和国个人信息保护法》《计算机软件保护条例》《计算机信息网络国际联网安全保护管理办法》等法律法规。任何违规使用所引发的民事赔偿、行政处罚或刑事责任，均由使用者独立承担。

应用管理是保障数字秩序的有力工具，但必须以合法、正当、必要为前提。请始终将尊重用户权利、遵守法律规范置于功能便利性之上。
